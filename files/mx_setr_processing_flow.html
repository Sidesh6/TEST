SAA-wrapped MX Message Structure Analysis
3-layer compound message. The parser must strip each layer and hand off cleanly to the next.
Layer 1 — SIBESHeader (Custom proprietary)
<SIBESHeader>
 <InputMessageReference>
  FDI0632200752000 ← FDA internal ref
 </InputMessageReference>
</SIBESHeader>

// Custom header — strip before XML parse
// Not valid XML part of the SAA PDU
Layer 2 — Saa:DataPDU / Saa:Header (SAA Envelope)
<Saa:DataPDU xmlns:Saa="urn:swift:xsd:saa.2.0"...>
 <Saa:Header>
  <Saa:Message>
   MessageIdentifier: setr.004.001.04
   Format: MX
   Sender DN: ou=funds,ou=tandt,o=testdeu1399,o=swift
   Sender X1 (BIC): CSAMCHZHXXX
   Receiver DN: ou=funds,ou=live,o=parbdeff,o=swift
   Receiver X1: PARBITMOXXX, X2: live

   Priority: Normal
   IsPossibleDuplicate: false

SAA Envelope Parser — Built In-House
A pre-parser that strips the compound wrapper and routes to the correct MX parser. Must run before any MX parsing.
1
SIBESHeader Stripper
Detect <SIBESHeader> prefix (not valid XML root). Extract InputMessageReference as internalRef. Strip the element — remaining bytes must be valid XML starting at <Saa:DataPDU>. Store internalRef in MessageContext.
2
SAA DataPDU Namespace Registration
Register all namespaces from root element: xmlns:Saa, xmlns:Sw, xmlns:SwGbl, xmlns:SwInt, xmlns:SwSec. Use StAX namespace-aware reader. Extract Saa:Revision (2.0.2) for version gating.
3
Saa:Header Extraction → SaaHeaderContext
Extract into SaaHeaderContext POJO:
• senderDN, senderBIC (X1), senderLogicalTerminal (X2)
• receiverDN, receiverBIC (X1), receiverEnvironment (X2: live/test)
• messageIdentifier (setr.004.001.04), format (MX), subFormat
• routingCode (FDI06), userReference, messageContext, messageNature
• priority, isPossibleDuplicate, service, network, sessionNr, seqNr
• swiftRef, snfQueueName, snfInputTime, snfDeliveryTime
4
Saa:Body Extraction
Navigate to <Saa:Body>. Extract child elements as separate DOM fragments: (1) AppHdr element, (2) Document element. Pass Document namespace URI to MX Parser for message type detection.
5
Route to MX Parser with full context
Call mxParserRegistry.getParser("setr.004.001.04") with the extracted Document bytes. Pass SaaHeaderContext as part of RawMessage metadata — available to all downstream stages via MessageContext.
public class SaaEnvelopeParser {
  SaaParseResult parse(RawMessage raw) {
    // 1. Strip SIBESHeader if present
    byte[] xmlBytes = sibesHeaderStripper.strip(raw.payload());
    // 2. Parse SAA envelope
    SaaHeaderContext saaCtx = saaHeaderExtractor.extract(xmlBytes);
    // 3. Extract Saa:Body Document bytes
    byte[] docBytes = saaBodyExtractor.extractDocument(xmlBytes);
    return new SaaParseResult(saaCtx, docBytes);
  }
}

public record SaaHeaderContext(
  String senderDN, senderBIC, receiverDN, receiverBIC,
  String messageIdentifier, routingCode, snfQueueName,
  boolean isPossibleDuplicate,
  String sessionNr, seqNr, swiftRef
) {}
